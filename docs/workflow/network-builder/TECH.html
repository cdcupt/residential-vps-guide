0Canonical interface — the single source of truth

The two worker sections independently sketched the engine seam and named a few things differently (estimate(state, roster) vs estimate(networkState); perEdge vs edges; detectGeo vs detectCountry). This section resolves those to exactly one contract. Everything below in §1 and §2 conforms to it; where a partial's SVG or prose still shows an older field name, this section wins.

▸ The whole contract — one pure fn, one async boot, one constant
Two calls and one object. estimate is pure and synchronous; the roster is a module constant (PROVIDERS), not a parameter; detectGeo is the only I/O on the entire surface.
// ── functions the view calls ──────────────────────────────────────────
estimate(state) → EstimateResult          // PURE · SYNCHRONOUS · call on every change
detectGeo({ timeoutMs = 1500 }) → Promise<{ country, source }>   // once, on load
PROVIDERS                                // shared roster constant (read-only) — feeds BOTH palette + estimate()

// ── the board state (the only input) ──────────────────────────────────
state = {
  geo:      'CN' | 'ROW' | null,      // detectGeo() country, normalized; null = unknown / lookup failed
  gatingOn: boolean,                // China guardrail; view derives DEFAULT from geo==='CN', user-overridable
  jump:     { provider, region, plan } | null,
  exits:    [ { provider, region?, plan?, traffic?: { pkgGB, cycle } } ]   // fan-out; duplicates allowed
}

// ── the return object the view renders (never recomputes) ─────────────
EstimateResult = {
  price: {
    monthlyUSD,                          // jump counted ONCE + Σ exits (flat + metered effective); refs excluded
    lines:      [ { label, usd, native? } ],   // one breakdown line per counted hop
    references: [ ... ],                 // reference / priced-only hops — listed, NEVER summed
    pending                              // a metered placeholder is folded in ⇒ panel stamps ⚑ph
  },
  latency:   { lo, hi, illustrative } | null,   // range across exits; illustrative = private-hop add
  io:        { iops, capped }          | null,   // min across exits (bottleneck); null ⇒ not benchmarked
  bandwidth: { mbps, pending }         | null,   // min across ALL hops (jump + every exit)
  route:     { label, purity, exitIPs, rankPending, benchmarked } | null,
  edges:     Edge[],                     // PRE-ORDERED: [entry, link×N]  OR  [entry, tail] (single-host)
  gating:    { active, entryBlocked, reason },   // drives entry-edge RED + notice
  estimated: true                       // invariant — ALWAYS
}

Edge = {
  kind:   'entry' | 'link' | 'tail',   // entry: you→jump · link: jump→exit · tail: jump→internet
  from, to,                              // display names, ready to print
  gia?,                                  // entry only — is the entry CN2 GIA?
  gated?,                                // entry only — gatingOn && !gia   (DECIDED IN THE ENGINE)
  latency: { lo, hi },
  bandwidthMbps,
  io:      { iops, capped },
  route:   { isp, rank, purity, rankPending },
  benchmarked,                           // false ⇒ "not benchmarked" (reference hop)
  exitIndex                              // 0-based; egress-IP label = exitIndex + 1
}

1Data model + estimate engine from TECH-backend

PROVIDERS.md is the source of truth, and it is deliberately region × plan shaped: BandwagonHost spans US-LA / SG / HK / JP with many plans; DMIT and VMISS likewise. So the model nests provider → region → plan, and the measurable facts (perf, latency, IOPS, bandwidth, route, purity) live on the plan — the unit we actually benchmarked. The palette shows one provider card; a placed jump node then grows a region + plan selector, so the model must let the UI enumerate regions and plans from data.

▸ Why plan-level facts, provider-level palette
A provider is one palette card, but its numbers differ per plan. The model puts metrics/pricing on the plan and lets the placed node carry {provider, region, plan}. Single-plan providers (SolaDrive, AaITR) collapse to a one-region, one-plan array — the UI reads them uniformly, so there is no special-case code path.

1.3 Estimate engine — the pipeline

One entry point — estimate(state) → EstimateResult — built from small pure functions: no globals, no DOM, no fetch, deterministic. The frontend calls it on every drop / remove / traffic-change and renders the returned object. The pipeline computes per edge first (connections are first-class), then folds edges into the five aggregate metrics.

1.4 The five formulas — exactly

#MetricFormulaNotes
①Price monthlyUSD = jumpUSD·1 + Σexit exitUSD jump counted once (shared relay). Reference boxes excluded (listed apart). Fold-in below.
②Latency [ minexit(jump.lo) … maxexit(jump.lo + exit.hopAdd) ] entry RTT measured; per-exit upper bound adds the illustrative private hop → a range. Single-host = jump's {lo,hi}.
③I/O minexit(exit.iops) bottleneck disk across the exits; capped if any contributor was throttled. Single-host = jump's iops.
④Bandwidth min( jump.bw, exit₁.bw, … exitₙ.bw ) bottleneck across all hops (jump + every exit) — the global cap.
⑤Route quality summarize( {isp,#rank}per exit ) + purity + N IPs identical exits collapse to "N× AT&T (#1)"; else joined. Purity common-or-"mixed". N = egress-IP count.

1.5 Price fold-in — flat, metered, multi-exit

// currency → USD (snapshot rates, refreshed manually)
const FX = { USD: 1, CAD: 0.73, CNY: 0.147 };
const toUSD = (amount, currency) => amount * (FX[currency] ?? 1);

// billing-cycle discounts (structure final; applies to metered packages)
const CYCLES = [
  { months: 1,  discount: 0.00 }, { months: 3,  discount: 0.05 },
  { months: 6,  discount: 0.10 }, { months: 12, discount: 0.20 },
  { months: 24, discount: 0.25 },
];

// effective monthly cost of a metered exit, in USD
function meteredMonthlyUSD(plan, pkgGB, cycleMonths) {
  const base = plan.pricing.packages[pkgGB];              // PM-pending placeholder ⚑ph
  const cyc  = CYCLES.find(c => c.months === cycleMonths) ?? CYCLES[0];
  return toUSD(base, plan.pricing.currency) * (1 - cyc.discount);
}

// whole-network price — jump counted ONCE, each exit summed, refs excluded
function priceUSD(jump, exits) {
  let total = 0; const references = [];
  if (jump && jump.pricing.type === 'flat') total += toUSD(jump.pricing.amount, jump.pricing.currency);
  else if (jump) references.push(jump);                  // metered/reference jump → not summed
  for (const e of exits) {
    if (e.pricing.type === 'flat')    total += toUSD(e.pricing.amount, e.pricing.currency);
    else if (e.pricing.type === 'metered') total += meteredMonthlyUSD(e, e.pkgGB, e.cycle);
    else                              references.push(e);   // reference → listed, not summed
  }
  return { monthlyUSD: total, references };
}

1.6 Geo-IP guardrail — client-side, keyless, graceful

The guardrail needs the visitor's country, client-side, with no backend and no secret. Primary is https://ipapi.co/json/; a second keyless source backs it up; total failure defaults gating off with the toggle shown. The toggle always wins — the user is authoritative. The view derives its gatingOn default from the returned country (=== 'CN').

// no key, no storage of the IP — only the country is read
async function detectGeo({ timeoutMs = 1500 } = {}) {
  const sources = [
    { url: 'https://ipapi.co/json/', pick: j => j.country_code },
    { url: 'https://ipwho.is/',      pick: j => j.country_code },
  ];
  for (const src of sources) {
    try {
      const ctl = new AbortController();
      const t = setTimeout(() => ctl.abort(), timeoutMs);
      const res = await fetch(src.url, { signal: ctl.signal });
      clearTimeout(t);
      if (!res.ok) continue;
      const code = src.pick(await res.json());
      if (code) return { country: code, source: src.url };   // e.g. "CN"
    } catch { /* timeout / network / CORS → next source */ }
  }
  return { country: null, source: null };   // → gatingOn=false, toggle shown
}

Privacy & security: no key, no request body, no cookie; we read only the country code and never persist the IP. This is the site's only runtime network call. A CSP connect-src allowance for ipapi.co + ipwho.is is the one edge-config note.

▸ One improvement over the prototype: delegate, don't re-wire
The DESIGN prototype re-binds listeners after every innerHTML swap. Production attaches one delegated listener per stable container (#nbx-board for click/keydown, the palette for dragstart, slots for drag/drop) reading data-* off the target. Re-rendering then only rewrites HTML — no re-binding, no leaked handlers. After a place(), focus moves to the just-placed node (or the AddExitTarget for an appended exit).

Actions are the only mutators — each mutates state, then calls render() and announce():

3.2 Decision B — the list-price rule

▸ Public price = official list price per (region, plan)
The public price for any box is its official list price for the selected (region, plan) from PROVIDERS.md. A benchmarked perf number pairs with its own plan's list price — no cross-pairing. No internal or discounted cost is ever shown publicly, so choose.html and the builder always agree.
