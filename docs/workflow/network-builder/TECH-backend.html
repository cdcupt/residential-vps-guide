1Provider data model

PROVIDERS.md is the source of truth, and it is deliberately region × plan shaped: BandwagonHost spans US-LA / SG / HK / JP with many plans; DMIT and VMISS likewise. So the model nests provider → region → plan, and the measurable facts (perf, latency, IOPS, bandwidth, route, purity) live on the plan — because that is the unit we actually benchmarked. The palette shows one provider card; a placed jump node then grows a region + plan selector (the same inline-configurator pattern as VirVM's traffic selector), so the model must let the UI enumerate regions and plans from data.

▸ Why plan-level facts, provider-level palette
A provider is one palette card, but its numbers differ per plan. The model resolves this by putting metrics/pricing on the plan and letting the placed node carry {providerId, regionId, planId}. Single-plan providers (SolaDrive, AaITR) collapse to a one-region, one-plan array — the UI still reads them uniformly, so there is no special-case code path.

2Module constants — currency & cycles

Two shared constants keep the money math honest and in one place (DRY). Rates are the same snapshots the fleet dashboard uses.

// currency → USD multiplier (snapshot "as of 2026-07-03"; refreshed manually)
const FX = { USD: 1, CAD: 0.73, CNY: 0.147 };
const toUSD = (amount, currency) => amount * (FX[currency] ?? 1);

// billing-cycle discounts (structure final; applies to metered packages)
const CYCLES = [
  { months: 1,  discount: 0.00, label: '1 month' },

3Estimate engine — the pipeline

One entry point — estimate(networkState) → EstimateResult — built from small pure functions: no globals, no DOM, no fetch, deterministic. The frontend calls it on every drop / remove / traffic-change and renders the returned object. The pipeline computes per edge first (connections are first-class), then folds edges into the five aggregate metrics.

4The five formulas — exactly

#MetricFormulaNotes
①Price monthlyUSD = jumpUSD·1 + Σexit exitUSD jump counted once (shared relay). Reference boxes excluded (listed apart). See fold-in below.
②Latency [ minexit(jump.lo) … maxexit(jump.lo + exit.hopAdd) ] entry RTT measured; per-exit upper bound adds the illustrative private hop → a range. Single-host = jump's measured {lo,hi}.
③I/O minexit(exit.iops) bottleneck disk across the exits; capped if any contributor was throttled. Single-host = jump's iops.
④Bandwidth min( jump.bw, exit₁.bw, … exitₙ.bw ) bottleneck across all hops (jump + every exit) — the global cap.
⑤Route quality summarize( {isp,#rank}per exit ) + purity + N IPs identical exits collapse to "N× AT&T (#1)"; else joined. Purity common-or-"mixed". N = egress-IP count.

① Price fold-in — flat, metered, multi-exit

// effective monthly cost of a metered exit, in USD
function meteredMonthlyUSD(plan, pkgGB, cycleMonths) {
  const base  = plan.pricing.packages[pkgGB];            // PM-pending placeholder ⚑ph
  const cyc   = CYCLES.find(c => c.months === cycleMonths) ?? CYCLES[0];
  return toUSD(base, plan.pricing.currency) * (1 - cyc.discount);
}

// whole-network price — jump counted ONCE, each exit summed
function priceUSD(jump, exits) {
  let total = 0; const refs = [];
  if (jump && jump.pricing.type === 'flat') total += toUSD(jump.pricing.amount, jump.pricing.currency);
  else if (jump) refs.push(jump);                       // metered/reference jump → not summed
  for (const e of exits) {
    if (e.pricing.type === 'flat')    total += toUSD(e.pricing.amount, e.pricing.currency);
    else if (e.pricing.type === 'metered') total += meteredMonthlyUSD(e, e.pkgGB, e.cycle);
    else                              refs.push(e);       // reference → listed, not summed
  }
  return { monthlyUSD: total, references: refs };
}

5Geo-IP guardrail — client-side, keyless, graceful

The guardrail needs the visitor's country, client-side, with no backend and no secret. The chosen primary is https://ipapi.co/json/; a second keyless source backs it up; total failure defaults gating off with the toggle shown. The toggle always wins — the user is authoritative.

default, not a lock. gate never blocks a build — it only recolours the entry edge + prints a notice.

Why ipapi.co (and why not the others)

CandidateVerdictReason
ipapi.co/json/ chosen · primary✓ Keyless for basic country lookup, HTTPS, sends Access-Control-Allow-Origin: * (works cross-origin from our non-CF site), returns country_code (ISO-2). We read one field.
ipwho.is/ chosen · fallback✓ Independent operator, also keyless + CORS + HTTPS, returns country_code. A second source guards against one host's rate limit / outage.
Cloudflare /cdn-cgi/trace✗ Two failures: (a) our site is not behind Cloudflare, so a same-origin /cdn-cgi/trace 404s; (b) the cross-origin www.cloudflare.com/cdn-cgi/trace sends no CORS headers, so a browser fetch is blocked.
ipinfo.io✗ Reliable country lookup increasingly wants a token — that reintroduces a secret/key we are told to avoid.
// no key, no storage of the IP — only the country is read
async function detectCountry({ timeoutMs = 1500 } = {}) {
  const sources = [
    { url: 'https://ipapi.co/json/',  pick: j => j.country_code },
    { url: 'https://ipwho.is/',       pick: j => j.country_code },
  ];
  for (const src of sources) {
    try {
      const ctl = new AbortController();
      const t = setTimeout(() => ctl.abort(), timeoutMs);
      const res = await fetch(src.url, { signal: ctl.signal });
      clearTimeout(t);
      if (!res.ok) continue;
      const code = src.pick(await res.json());
      if (code) return { country: code, source: src.url };  // e.g. "CN"
    } catch { /* timeout / network / CORS → next source */ }
  }
  return { country: null, source: null };  // → gatingAuto=false, toggle shown
}

Privacy & security: no key, no request body, no cookie; we read only the country code and never persist the IP. The lookup is the site's only runtime network call — everything else is the embedded constant. (A CSP connect-src allowance for ipapi.co + ipwho.is is the one edge-config note for the frontend/ops side.)

6Honesty invariants — enforced in data & logic

The guide's standing principle — never present a number as measured unless it is — is encoded, not just styled:

InvariantHow the logic enforces it
Everything is "estimated"EstimateResult.estimated === true always; the panel label is not optional.
Benchmarked vs referencereference plans have metrics === null ⇒ engine emits "not benchmarked" for perf/latency/io/bw/route and excludes them from the price sum (listed under references[]).
No fabricated numbersa missing measurement is null, never a guess. null renders as "not benchmarked" / "pending", never as 0 or an invented value.
PM-pending stampedillustrative/awaited values carry a boolean flag — route.rankPending, pricing.packagesPending, latency's pending (private-hop add) — which the frontend renders as ⚑ph.
VirVM traffic $ are placeholderspricing.packages values are placeholders behind packagesPending:true; they fold into the total (so the UX works) but are visibly stamped and swapped for real numbers when the PM supplies them.
Currency truthnative amount + currency are preserved; only toUSD() output is summed, and the native value is shown beside the ≈USD.
Stock is orthogonalstock:'out' shows a badge but the box is still priced (DMIT/VMISS/QQ.pw) — honesty about availability, not suppression.

7Interface exposed to the frontend

The backend layer exposes data + pure functions; the frontend owns all DOM. Two calls and one object are the whole contract.

// ---- functions the frontend calls ----
estimate(networkState) => EstimateResult          // pure; call on every change
detectCountry({timeoutMs?}) => Promise<{country, source}>   // once, on load
PROVIDERS                                          // the embedded roster constant (read-only)

// ---- the return object the frontend renders ----
EstimateResult = {
  price: {
    monthlyUSD: number,                            // jump ×1 + Σ exits (flat + metered eff.)
    lines: [{ label, amountUSD, native?:{amount,currency}, kind }],
    references: [string],                          // reference boxes, not summed
    pending: boolean                               // a metered placeholder is involved
  },
  latency:   { loMs, hiMs, pending } | null,        // range across exits; pending = hop-add illustrative
  io:        { iops, capped, pending } | null,      // bottleneck; null ⇒ not benchmarked
  bandwidth: { mbps, pending } | null,             // min across hops
  route:     { label, purity, exitCount, rankPending,
               isDatacenter, notBenchmarked } | null,
  egress:    { ipCount },                          // N separate exit IPs (0 = single-host DC egress)
  edges: [                                         // per-edge readouts, in order
    { kind:'entry', from, to, latencyMs, gia, gatedRed },
    { kind:'link',  from, to, latencyMs, bandwidthMbps, iops, iopsCapped,
      route:{isp,rank,rankPending,purity}, egressIp:n }
  ],
  gating: { active, entryBlocked, reason },        // drives edge① red + notice
  estimated: true                                // invariant
}
▸ Contract guarantees
1 Pure & synchronous — estimate() does no I/O and returns the same object for the same state (safe to call on every keystroke/drop). 2 Total-safe — an empty board, a single-host network, and an all-reference board each return a valid object (nulls, not throws). 3 The frontend never recomputes money/latency itself — it renders these fields verbatim, so the two sections cannot drift.

8Integration & verification

  • Embedded, not fetched. PROVIDERS ships as a JS constant inside tool.html (curated snapshot, "as of" date), matching the static-site / no-backend / manual-refresh constraint. Geo-IP is the only runtime fetch.
  • Deployment = the file ships. No server, no build step, no DB. The logic layer is a <script> block, styled by the existing style.css and .perf-ref tokens.
  • choose.html Value column is a pure derivation off the same roster: Value = metrics.perf ÷ monthlyUSD (pt/$), rendered "—" for metered until a package is chosen. No new data.
  • Reference implementation exists. DESIGN v3's prototype (metrics(), roster R, presets) is the flat-shaped seed; the production model adds the region×plan nesting and the resolver, keeping the same math.
  • Responsive: every diagram sits in an overflow-x:auto figure with svg{max-width:100%}; verified no page-level horizontal overflow at 1440 / 390 / 320 (see below).
