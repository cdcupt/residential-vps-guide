2Board anatomy

3One affordance, three inputs accessibility · non-negotiable

The central accessibility decision: a palette card is a native <button> that is also draggable. Its button semantics are the baseline that touch and keyboard use; drag-and-drop is a progressive enhancement layered on top. No path is exclusive, and drag is never the only way to do anything.

Accessibility specifics

  • Roles & labels. Palette = a labelled list of buttons (aria-label="Add BandwagonHost as jump host"). Slots are role="group" with an accessible name ("Jump host, empty" / "Jump host: BandwagonHost").
  • Live region. An off-screen aria-live="polite" node announces every place / remove / gating change ("Added SolaDrive as exit. Estimated cost $70 per month.").
  • Keyboard remove/config. A placed card's ✕ and its traffic chips are all focusable buttons; Delete on a focused placed card removes it.
  • Focus visible & reduced-motion. 2px accent outline on every interactive element; drag ghosting and slot pulse honour prefers-reduced-motion.
  • Touch targets. Cards and chips are ≥ 40px tall; the whole card is the hit area, not a tiny handle.

5China CN2-GIA guardrail

▸ Toggle copy & behaviour
Bar reads: "Building from mainland China? Auto-detected: yes · your route to China is gated to CN2 GIA." with a switch. Flipping it off relabels to "Gating off" and clears red states. The detected value seeds the switch but the user always wins. Detected country shows as a muted geo: CN (auto) stamp so the decision is transparent, never silent.

6Traffic-metered pricing prices PM-pending

When a metered provider (VirVM on its 专线/CN2-GIA residential plan) lands on the canvas, its card grows an inline selector — it does not open a dialog. Two controls fold into one monthly figure:

ControlOptionsEffect on cost
Traffic packagechips: 100 · 300 · 500 · 1000 GBsets the package base price for the chosen tier (PM-pending numbers).
Billing cycleselect: 1 mo · 3 mo −5% · 6 mo −10% · 1 yr −20% · 2 yr −25%applies the cycle discount, shown as an effective $/mo so it compares to flat boxes.

The estimate panel then shows the fold-in explicitly, e.g. 300 GB pkg $28 × 12 mo −20% = $268.80/yr → $22.40/mo effective, summed with any flat hops. Placeholder prices in the prototype are visibly stamped placeholder so nobody mistakes them for measured data. The tier + cycle structure is final; only the dollar amounts wait on the PM.

⛁ What the build needs from the PM
For each metered provider: the base price per package — VirVM: 100GB=$?, 300GB=$?, 500GB=$?, 1000GB=$? — and whether the cycle discounts (−5/−10/−20/−25%) are provider-wide or per-package. The data model already carries these as fields; the UI renders whatever numbers arrive.

8Cost-performance on choose.html

One derived column joins the existing fleet .perf-ref table — Value = Overall ÷ $/mo, in "points per dollar." No new measurement; it's arithmetic on the two columns already there. The best-value row is tinted teal with a ★ best value badge, and the column is sortable. Mock of the augmented table:

The metric is deliberately blunt and explainable — highest performance-per-dollar wins — so a value shopper reads it at a glance, exactly the gap the PRD calls out. It reuses .perf-ref tints (.hi/.lo language) so it looks like it was always there. Metered providers can't yield a single $/mo, so they show "—" and defer their value to the builder where a package is chosen.

9Open items for the PM

  1. price Traffic-package base prices for VirVM (and any other metered provider): 100 / 300 / 500 / 1000 GB, plus the base, and whether cycle discounts are provider-wide or per-package.
  2. roster Additional palette providers (name · link · specs · flat price or reference). The palette is data-driven and will render whatever roster ships.
  3. decision Multi-exit scope for v1 — the model supports 0…n exits; do we expose "＋ add another exit" in v1 or ship 0-or-1 first? (Design covers both; recommend 0-or-1 visible, n-ready underneath.)
  4. decision Geo-IP provider — which free client-side lookup (e.g. a Cloudflare trace or a free JSON endpoint), and the fallback copy when it's unreachable.
  5. copy Value column tooltip — confirm the one-line explanation of "pt/$" shown on hover/tap for the Choose table.

10Accessibility & responsive verification

  • Widths checked: 1440 (desktop three-column), 390 (mobile stack), 320 (smallest) — no horizontal overflow; wide SVGs and tables live inside overflow-x:auto containers.
  • Contrast: text on rust/teal/indigo backgrounds uses the darker -ink token pairs (WCAG AA), matching the site's own convention.
  • Keyboard: every place / remove / toggle / chip is a focusable control with a visible 2px focus ring; nothing is drag-only.
  • Motion: drag ghost + slot highlight collapse to instant under prefers-reduced-motion.
  • Honesty: estimated values tagged; benchmarked vs reference badged; placeholder traffic prices stamped — no number is presented as measured unless it is.
