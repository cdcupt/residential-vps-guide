VPS Network Builder & Cost-Performance

A drag-and-drop board where a user assembles their own VPS network from real providers and sees estimated performance, latency and cost — with mainland-China CN2-GIA guardrails — plus a real cost-performance figure on every VPS in the guide.

residential-vps.daichenlab.com · Orchestrator-authored · Gate target: Gate 1 (Design)

1Problem & goal

The guide already measures VPS boxes and lists prices, but a first-time builder still has to hold it all in their head: which box is a good deal, whether the route works from China, and what a two-box network will cost and perform like. Two gaps:

  • No value-for-money signal. Perf and price sit in separate columns — nobody computes "performance per dollar."
  • No way to try a network on. The current builder is a fixed pair of radio lists; it can't express "drag these together and tell me what I get."

Goal: let a user assemble a network by dragging real providers onto a board and instantly see estimated performance, latency and cost — and surface a cost-performance number on every VPS — so that choosing and composing a China-ready VPS network becomes a two-minute, self-serve task.

3Scope

Out of scope (v1): user accounts / saved builds; automatic ordering/checkout; a backend or database; live stock/price polling (prices are curated snapshots we refresh); benchmarking new providers (reference-only until we deploy + measure).

4The board — product model

Estimate rules (all labelled "estimated"):

  • Performance — from the exit box's measured score when present, else the jump host's; reference-only boxes show "not benchmarked."
  • Latency — the jump host's measured China RTT (entry), noting the exit adds a private hop. Real measured numbers for our fleet; reference boxes marked.
  • Cost — flat $/mo summed across hops, plus any traffic package for metered providers (see §6).

8Success criteria

  1. A user can drag a jump host (and optionally exits) onto the board and see a live perf · latency · cost estimate in ≤ 2 interactions.
  2. Every VPS on choose.html shows a correct, derived cost-performance value (perf ÷ $/mo), and the best value is visible at a glance.
  3. With China gating on, a non-CN2-GIA entry hop is unmistakably red and a notice explains why.
  4. A traffic-metered provider's cost updates correctly when the user picks a 100/300/500/1000 GB package and a billing cycle.
  5. Estimates are clearly labelled estimated; benchmarked vs reference boxes are visually distinct; no fabricated numbers.
  6. Works with zero backend, no horizontal overflow, legible at 1440 / 390 / 320.

9Constraints, risks & open items

  • Static site, client-side only. Geo-IP via a free third-party lookup with graceful fallback (if it fails, gating defaults off and the toggle is shown). No secrets, no backend.
  • Honesty guardrail (standing site principle). Only measured numbers for boxes we run; reference providers are price/spec only and badged. Never fabricate perf, latency, or price.
  • Drag-and-drop accessibility. Must have a keyboard/tap fallback (click-to-place) so it works on touch + without a mouse.
  • Data freshness. Prices are curated snapshots (with an "as of" date), not live — refreshed manually.
  • OPEN PM to supply the additional-provider list and the traffic-package prices (§6 callout).
